Security

ADAMZ applies practical controls to protect your data workflows and account operations.

Platform safeguards

  • HTTPS/TLS transport for data in transit between browser, API, and storage providers.
  • Server-side payment confirmation with Stripe session validation before upload unlock.
  • Runtime configuration via environment variables to avoid exposing secret keys in frontend code.
  • Dependency updates and security patching as part of release maintenance.

Operational controls

  • Transaction history records with timestamps to support traceability and incident review.
  • Access-protected history page credentials managed at API level.
  • Storage upload signing flow to avoid direct long-term credential exposure in the browser.

Customer security best practices

  • Use strong, unique passwords for deployment and admin endpoints.
  • Set strict `ALLOWED_ORIGINS` in production and avoid wildcard CORS policies.
  • Rotate service keys periodically and immediately after team or environment changes.
  • Limit uploaded datasets to minimum-required fields for analytics.

Incident reporting

If you identify a security concern, contact support with reproduction steps, impacted URL, and timestamp so we can prioritize triage.
